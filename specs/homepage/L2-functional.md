# L2: 機能仕様 - ホームページ

## ページ構成

```
┌─────────────────────────────────────┐
│  ヘッダー（固定）                    │
├─────────────────────────────────────┤
│  ヒーローセクション                  │
├─────────────────────────────────────┤
│  アプリセクション 1                  │
├─────────────────────────────────────┤
│  アプリセクション 2                  │
├─────────────────────────────────────┤
│  アプリセクション 3                  │
├─────────────────────────────────────┤
│  フッター                           │
└─────────────────────────────────────┘
```

---

## 1. ヘッダー

### 表示要素
- ロゴ（左寄せ）
- ナビゲーションメニュー（右寄せ）
  - About Us
  - Contact（将来追加予定）

### 振る舞い
- 画面上部に固定表示（スクロールしても追従）
- 背景は半透明または背景色に馴染む色

### レスポンシブ
- **PC**: ロゴ + 横並びメニュー
- **スマホ**: ロゴ + ハンバーガーメニュー
  - タップでメニューが展開（オーバーレイ）

---

## 2. ヒーローセクション

### 表示要素
- キャッチコピー（大きな文字）
- サブテキスト（説明文）
- 背景（グラデーションまたは画像）

### 振る舞い
- ファーストビューで全画面表示（100vh）
- スクロールを促すインジケーター（下矢印アニメーション等）

---

## 3. アプリセクション

### 表示要素（1セクションあたり）
- アプリ名
- アプリの説明文（1〜2文）
- アプリのイメージ画像またはアイコン
- アクセスボタン（リンク）

### 振る舞い
- スクロールに応じて要素がフェードイン
- 背景色がセクションごとに変化（滑らかなトランジション）
- 画像は視差効果（パララックス）または滑らかなアニメーション

### スクロールアニメーション
- 要素が画面内に入ったタイミングでアニメーション開始
- フェードイン + 軽い上方向への移動
- 背景色は前のセクションから次のセクションへグラデーション遷移

### 初期データ（モック）
| アプリ名 | 説明 | リンク先 |
|---------|------|---------|
| App 1 | サンプルアプリ1の説明 | https://google.com |
| App 2 | サンプルアプリ2の説明 | https://google.com |
| App 3 | サンプルアプリ3の説明 | https://google.com |

### 表示要素
- コピーライト
- SNSリンク（将来追加予定）

## 5. デザイン仕様

### カラーパレット
| 用途 | 色 | 備考 |
|------|-----|------|
| プライマリ | 暗めのワインレッド（#722F37 程度） | メインアクセント |
| 背景（メイン） | 黒〜ダークグレー（#0a0a0a〜#1a1a1a） | ベース |
| 背景（セクション） | ワインレッド〜黒のグラデーション | セクションごとに変化 |
| テキスト | 白（#ffffff） | 本文 |
| テキスト（サブ） | ライトグレー（#a0a0a0） | 補足テキスト |

### タイポグラフィ
- 見出し: 大きく、太く、インパクト重視
- 本文: 読みやすいサイズ（16px以上）
- 行間: 1.6〜1.8

### アニメーション
- 滑らかさ重視（ease-out系のイージング）
- 過度なアニメーションは避ける
- 60fps維持を目標

---

## 6. レスポンシブ対応

### ブレークポイント
| デバイス | 幅 |
|---------|-----|
| スマホ | 〜767px |
| タブレット | 768px〜1023px |
| PC | 1024px〜 |

### スマホ版の調整
- ヘッダー: ハンバーガーメニュー
- アプリセクション: 縦積みレイアウト
- フォントサイズ: やや小さめに調整
- アニメーション: 軽量化（パフォーマンス考慮）

---

## 7. アクセシビリティ

- キーボードでの操作が可能
- スクリーンリーダー対応（適切なaria属性）
- 色のコントラスト比 4.5:1 以上
- アニメーション軽減設定（prefers-reduced-motion）対応
